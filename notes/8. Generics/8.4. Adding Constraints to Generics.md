# Adding Constraints to Generics

Suppose you have a generic `T`, but you want it to only be a string or a number, not anything else.

You can add constraints with the `extends` property. 

> **Note:** Here `extends` isn't the inheritance extends, it just means "this must satisfy...".

```typescript
function doSome<T>(value: T): T {
    return value;
}
```

This works for all values.

But if I only want a generic type variable that should be *at least* a string:

```typescript
function doSome<T extends string>(value: T): T {
    return value;
}   
```

This just means `T` is a type variable that satisfies `string` (so `T` must be a string).

```typescript
doSome('hello') // works fine
doSome(12) // ERROR: type number isn't assignable to string or whatever ....
```

### Other Constraints

Suppose you want a generic to be of only a type where it has a `length` property.

```typescript
function getLength<T extends { length: number }>(value: T): T {
    return value.length;
}
```

`<T extends { length: number }>` just means, "T is a type variable, which is an object, and must have a property `length` which is a number."

> **Gotcha:** Strings and arrays work too, cause they have `length` and any type in JS is basically an object under the hood. So yeah.

---

> **Heads up:** With generics, it's just saying "I don't know the type yet, but I want TypeScript to remember the type of what I pass."

# Type Assertions

Basically, the "Trust me bro" analogy.

It's used when you know more about the data than TypeScript can infer. When you are sure about it.

Suppose you have a union type:
```typescript
let a: string | number;
```

If you do `a.` and look for IntelliSense suggestions, you won't see any until you type guard or narrow it down with `if` checks.

If you know that the union type variable is definitely a string, you can force TS to believe that by using the `as` operator.

```typescript
(a as string).___ // you get shown string properties and methods by IntelliSense
```

Another syntax for type assertion:
```typescript
<string>a.___
```

This works with `unknown` types too.

# Type Casting

Basically converting one datatype to another.

Like:
```typescript
let a: string = '12';
Number(a); // this is type casting, we are converting string to number
```

That's it.

# Basic Types

### Primitive Types
Simple types that can be directly duplicated in memory.
```typescript
let a = 1;
let b = a;

b++;
// Only b changes to 2, a remains 1
```
In memory, `a` holds `1` and `b = a` copies `1` to `b` directly.

### Reference Types
Arrays, Objects, etc. They can't be copied/duplicated easily.
```typescript
let a = [1, 2, 3];
let b = a;

b.push(4);
// Both a and b are now [1, 2, 3, 4]
```
`a` holds a reference (pointer) to the array in memory. When you copy `b = a`, `b` gets the same reference. Modifying `b` changes the shared array.

## TS Types & Definitions

### Primitives
```typescript
let a: number = 12.42;
let b: string = 'hello';
let c: boolean = true; // Note: Use 'boolean', not 'bool'
```

### Arrays
```typescript
let numArr: number[] = [1, 2, 4]; // Errors if you put anything other than numbers

// Union type (mixed data array)
let mixArr: (string | number)[] = ['a', 1, 2, '3'];
```

### Tuples
An array with a fixed number of elements and specific types based on their position. Not like Python tuples.
```typescript
let tup: [number, string, number] = [1, 'hello', 2];
```
> [!WARNING]
> **Tuple Gotcha:** You can still use array methods like `.push()` or `.pop()` on tuples, and TS won't catch it at compile-time (e.g. `tup.push(3)` works).

### Enums
Short for enumeration. Set of predefined constants instead of arbitrary values. (Use `=` to assign values, not `:` like objects).
```typescript
enum roles {
  ADMIN = 'admin',
  GUEST = 'guest',
}

enum currency {
  USD = '$',
  EUR = '€',
  NPR = 'रू.',
}

// Access like an object:
console.log(currency.USD); // '$'
```
> [!TIP]
> **Enum Gotcha:** If you don't assign values, they auto-increment from `0` (e.g. `enum Direction { Up, Down }` -> `Up` is `0`, `Down` is `1`).

## Other Types

### any
Can be anything. Disables all TS compiler checks for that variable. 
If you define a variable without a type or value, TS defaults it to `any`:
```typescript
let a; // type is any
```
> [!CAUTION]
> **Avoid any.** It is an escape hatch and defeats the purpose of TypeScript.

### unknown
Similar to `any`, but safer. TS locks it so you can't access methods or properties until you verify the type.
```typescript
let a: unknown = 'This is a string';

a.toUpperCase(); // ERROR: 'a' is of type 'unknown'
```
To unlock it, perform a type check:
```typescript
if (typeof a === 'string') {
  a.toUpperCase(); // Allowed! TS knows it's a string here
}

// Ternary example
typeof a === 'number' ? a.toFixed(2) : a;
```

### void
Represents the absence of any value. Usually used for function return types that don't return anything.
```typescript
function log(msg: string): void {
  console.log(msg);
}
```

### null & undefined
Usually used with union types to handle empty values.
```typescript
let user: string | null = 'val';
```

### never
Represents values that should never occur. Blocks all code after it from running.
```typescript
// Example: functions that throw errors or infinite loops
function throwError(msg: string): never {
  throw new Error(msg);
}
```

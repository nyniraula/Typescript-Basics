# Type Guards or Narrowing

This just means when you have a union type or `unknown` type, you can't access its specific properties without verifying the type first.

Here is how you do that:

### 1. `typeof`

```typescript
let a: string | number;

if (typeof a === 'string') {
    a.toUpperCase(); // works
}
```

### 2. `in` operator

Suppose you have two types, `User` and `Admin`:

```typescript
type User = {
    name: string;
}

type Admin = {
    name: string;
    permissions: string[];
}
```

You have a `person` object. You want to execute permissions by checking if they are an admin. How?

```typescript
// checking if a key exists
if ('permissions' in person) { 
    person.permissions;
}
```

### 3. `instanceof`

Basically used to check if an object is an instance of / belongs to that class.

```typescript
const date = new Date();

if (date instanceof Date) {
    date.getFullYear();
}
```

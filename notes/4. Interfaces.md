# Interfaces

Defines the shape of an object in TypeScript.

Annotate an object with an Interface so TS knows what properties it should have. Gives inline suggestions for keys, and throws errors for unknown keys or incorrect types. Mainly used for defining function params, but sometimes normal objects too.

## Syntax & Example
```typescript
interface User {
    name: string;
    age: number;
    isAdult: boolean;
}

const Adam: User = {
    name: 'Adam',
    age: 18,
    isAdult: true
};

// Error Example:
const InvalidUser: User = {
    name: 'Adam',
    age: '18',     // Error: Type 'string' is not assignable to type 'number'
    isAdult: 1,    // Error: Type 'number' is not assignable to type 'boolean'
    extraKey: true // Error: Object literal may only specify known properties
};
```
If an object misses any of the keys or includes extra keys, TS throws an error.

---

## Optional Values
A value that may or may not be in the object. Put a question mark (`?`) before the colon.

```typescript
interface User {
    name: string;
    age: number;
    email: string;
    gender?: string; // Optional value
}
```

---

## Extending Interfaces
Basically like inheritance. You can extend an interface from another and inherit its keys.

```typescript
interface Guest {
    name: string;
    email: string;
    pin: number;
}

interface Admin extends Guest {
    isSuperAdmin: boolean;
}

// When you do adminObj. , TS suggests all Guest keys and checks their types too.
```

---

## Merging Interfaces
If you declare two interfaces with the same name, TS merges them.

```typescript
interface User {
    name: string;
}

interface User {
    age: number;
}

// TS merges them into:
// interface User { name: string; age: number; }
```

---

## Interfaces in Params
The main use of interfaces is checking shapes passed into functions.

```typescript
interface User {
    name: string;
    age: number;
}

function getUserData(obj: User) {
    // typing obj. will trigger suggestions for name and age
    return obj.name; 
}
```

### Abstract Classes and Methods

Abstract classes act as blueprints for other classes. They cannot be instantiated directly with `new`; they must be inherited.

They are used to enforce and share behavior among subclasses. You declare the properties or methods that should exist without providing their actual implementation.

#### Syntax

```typescript
abstract class ClassName {
    abstract fn(): return_type;
}
```

#### Example Scenario

Imagine a payment API supporting PayPal, Stripe, and Wise. Each has a different underlying implementation, but they all need a standard way to process payments.

```typescript
abstract class Payment {
    abstract get_payment(): void; // Signature only, no body
}

class Paypal extends Payment {
    get_payment() {
        // PayPal-specific implementation
        console.log("Processing PayPal payment...");
    }
}

class Stripe extends Payment {
    get_payment() {
        // Stripe-specific implementation
        console.log("Processing Stripe payment...");
    }
}
```

> [!IMPORTANT]
> **Gotchas**:
> 1. You cannot instantiate an abstract class directly: `new Payment()` will throw a compiler error.
> 2. Any subclass that extends an abstract class **must** implement all of its abstract methods, otherwise TS will throw an error.

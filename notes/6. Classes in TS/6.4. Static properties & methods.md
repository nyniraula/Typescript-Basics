### Static Properties and Methods

Static properties and methods belong to the class itself rather than instances of the class. They can be accessed directly without instantiating the class.

> Think of built-in helpers like `Math.PI` or `Math.random()`. You don't initialize a `new Math()`; you just call them directly.

Use the `static` keyword to define them:

```typescript
class API {
    static version = '2.20.1';

    static getAbout() {
        return "This is an API about X";
    }
}

// Accessed directly on the class
console.log(API.version);    // "2.20.1"
console.log(API.getAbout()); // "This is an API about X"
```

#### Mixing Static and Instance Members

Classes can have both static and non-static members:

```typescript
class API {
    static version = '2.20.1';

    static getAbout() {
        return "This is an API about X";
    }

    constructor(public data: string) {}

    produceResult() {
        return this.data + " processed";
    }
}
```

> [!NOTE]
> When you instantiate the class with `new API("my-data")`, the instance will only have the non-static properties/methods (like `data` and `produceResult`). It will not contain `version` or `getAbout`.

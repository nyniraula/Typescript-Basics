### Parameter Properties

Declaring properties at the top, then in the constructor arguments, and then mapping `this.variable = variable` is too much boilerplate.

```typescript
// The old boilerplate way
class Human {
    public name: string;
    private gender: string;
    protected age: number;

    constructor(name: string, gender: string, age: number) { 
        this.name = name;
        this.gender = gender;
        this.age = age;
    }
}
```

TS solves this with **parameter properties**. You declare the properties directly inside the constructor arguments by prefixing them with access modifiers. TS automatically creates the properties and assigns the values behind the scenes.

```typescript
// The clean TS way
class Human {
    constructor(
        public name: string, 
        private gender: string, 
        protected age: number
    ) {
        // No boilerplate fields or this.x = x assignments needed!
    }
}
```

---

### Optional Properties

You can make properties optional by adding a `?` before the type annotation.

```typescript
class Human {
    constructor(
        public readonly name: string, 
        protected age: number, 
        private gender?: string // Optional parameter
    ) {}
}
```

> [!IMPORTANT]
> **Gotcha**: You **must** explicitly write the access modifier (`public`, `private`, `protected`, or `readonly`) in the constructor parameters. If you omit it, TS will treat it as a regular constructor argument and won't auto-create the class property.

### Getters and Setters

You can define getter and setter methods to access and modify private properties as if they were public fields, allowing you to intercept reads and validation on writes.

```typescript
class Account {
    constructor(private _balance: number) {}

    // Getter
    get balance(): number {
        return this._balance;
    }

    // Setter
    set balance(value: number) {
        if (value <= 0) {
            throw new Error("Value can't be less than 0");
        }
        this._balance = value;
    }
}
```

> **Why the underscore (`_balance`)?**
> It's a convention. The actual property needs a different name than the getter/setter methods (`balance`), otherwise you'll get naming collisions and infinite loops during recursion.

#### Usage

Getter and setter methods are called as if they are normal properties, without parenthesis:

```typescript
let user = new Account(2000);

// Calls the getter
console.log(user.balance); // 2000

// Calls the setter
user.balance = 500; 
```

> [!NOTE]
> **Gotcha**: 
> 1. If a property has a `get` but no `set`, it is automatically inferred as `readonly`.
> 2. Getters/setters require the compilation target to be ES5 or higher in `tsconfig.json`.

# Types & Type Aliases

Custom mixed types from type aliases.

### Syntax
```typescript
type NameOfType = number | null;
```

### Union Types
```typescript
type StrNull = string | null;
```
The pipe (`|`) is called a union—it just says "this or that".

### Intersection Types
Combine two types using ampersand (`&`).

Suppose you have two types used in different places:
```typescript
type User = {
  name: string;
  email: string;
};

type TimeStamp = {
  createdAt: Date;
  updatedAt: Date;
};
```

Combine them into a new type:
```typescript
type UserRecords = User & TimeStamp;
```

### Type vs Interface
Sounds similar to merging or extending interfaces?
Here's the catch:

* **Merging interfaces:** Declaring the same interface twice merges them into one. Types can't be merged/re-declared.
  ```typescript
  interface User { name: string; }
  interface User { email: string; } // Merges into one User type with both properties
  ```
* **Extending interfaces:** Interfaces use `extends` to inherit from one or more interfaces at once.
  ```typescript
  interface User { name: string; }
  interface TimeStamp { createdAt: Date; }
  interface UserRecords extends User, TimeStamp {}
  ```

### Why intersect (`&`) instead of extend?

1. **Working with Unions:** Interfaces cannot extend union types. You must use intersection instead.
   ```typescript
   type Status = "active" | "inactive";
   type UserStatus = Status & { lastActive: Date }; // Works
   ```

2. **Inline / Anonymous Types:** You can intersect types on the fly (e.g., in function parameters) without needing to declare a new interface name.
   ```typescript
   function saveUser(data: User & TimeStamp) {}
   ```

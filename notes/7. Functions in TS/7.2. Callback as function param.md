# Callback as function param

What is a callback?

Callback is just a function that you pass as an argument to another function. (That function might run it inside itself or return it in some manner).

Basic example of callback in JS:

```javascript
function runCallback(fn) {
    fn()
} 

runCallback(() => { console.log('haha') }) // just passing an anonymous function as argument
```

Example of callback that also has its params:

```javascript
function runCallback(value, fn) {
    fn(value)
}

runCallback(20, (data) => { console.log(data) })
```

Here what happens under the hood is we pass a function that has `data` params. When it reaches `runCallback`, it acts as an arrow function variable:

`fn = (data) => { console.log(data) }`

Then we just call it.

--- 

Now in TS:

```typescript
function runCallback(fn: (params: type) => return_type): void {
    // ...
}
```

eg:

```typescript
function runCallback(value: number, fn: (data: number) => number): void {
    // ...
}
```

> **Type Signatures**: In TS, the type of a function is written as `(paramName: ParamType) => ReturnType`. The parameter names in the signature don't have to match the names when you actually call or define the callback, but they help with readability.

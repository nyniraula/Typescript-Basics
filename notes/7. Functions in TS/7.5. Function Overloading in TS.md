# Function Overloading in TS

What is function overloading?

It's basically a concept where a function does multiple things based on diff params and types of params. The function has the same name but diff implementation based on params and types.

JS or TS don't natively support function overloading, we just kind of mimic it by type checking and stuff.

Eg of real func overloading in C++ to clarify:

```cpp
void greet() {
    cout << "Hello\n";
}

void greet(string name) {
    cout << "Hello, " << name << '\n';
}

void greet(string name, int age) {
    cout << "Hello, " << name << ", age " << age << '\n';
}

greet(); // executes the first func
greet("Val"); // executes the second func
greet("Val", 18); // executes the third func
```

In TS and JS, the only way to do this is by making params optional and checking if a param is undefined to execute one or the other.

First we define the signatures (type declarations) of the functions before implementing it if we wanna overload it in TS.

```typescript
function greet(): void;
function greet(name: string): void;
function greet(name: string, age: number): void;
```

Then we define a single implementation function with all params as optional and check on the basis of them.

```typescript
function greet(name?: string, age?: number): void {
    if (name === undefined) {
        console.log('Greet')
    } else if (age === undefined) {
        console.log(`greet ${name}`)
    } else {
        console.log(`greet ${name} - ${age}`)
    }
}
```

Basically trying to mimic the feature from C++ / Java.

> **Gotcha**: The implementation signature is *not* visible from the outside. Only the overload signatures are used by TS to check your function calls.

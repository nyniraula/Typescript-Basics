# Functions basics in TS

Lets recap some js:

In js we declare functions as:
```javascript
function fn_name(params) {
    // some things
    return some_value // (optional)
}

// or 

const fn_name = (params) => {
    // some things
    return value
}
```

In TS its the same with slight annotations:

```typescript
function fn_name(params: type): return_type {
    // do some stuff
    return // or not
}

// or for arrow funcs:

const fn_name = (params: type): return_type => {
    // do some stuff
    return // value or not
}
```

eg:

```typescript
function logTotal(a: number, b: number): void {
    console.log(a + b)
}

// or 

const calcTotal = (a: number, b: number): number => {
    return a + b
}
```

> **Note / Warn**: in TS, you have explicitly state no return as `void` too.
